와이스 신규 설치 유저들에게 앱 초기 화면에서 해당 유저가 선택한 옵션에 따라 자동적으로 라이브 딜러를 추천하는 온보딩 페이지 개발을 검토 중.

하단 라벨링 목록과 Funnel 폴더 내의 각 파일을 참고해 각 딜러별 최근 라이브, 최근 판매 내역, 라이브 빈도, 활동 시간대를 확인한 뒤 적절히 라벨링을 진행

## 매칭 규칙

- 최신 내역일수록 더 높은 가중치를 적용
    - 0–7d ×4.0, 7–14d ×2.5, 14–30d ×1.0, 30–90d ×0.5, 90–180d ×0.25, 180d+ ×0.1
- 상품보다 라이브에 좀 더 높은 가중치 적용
    - 라이브 = 상품 10개 수준으로 취급
- 상품명이나 라이브명만으로 라벨링이 어려운 항목은 제외
- 라이브명에 [테스트]가 포함되어 있거나, 판매 금액이 0인 라이브는 집계에서 제외
- 주라벨 키워드가 컷오프에 걸리면 부라벨로 이동
    - 주라벨 기준: 가중 ≥30, 점유율 ≥8%, 상위 3개만 유지 · 탈락분은 부라벨 승계
- 상품명 끝에 "피규어"가 접미사로 붙을 경우, 모두 [일반]으로 분류
- 중복 상품 처리: '#숫자_라이브명' 형태로 반복되는 판매는 1건으로 집계
    - 와이스의 상품명은 딜러가 별도로 직접 입력하지 않으면 '#숫자_라이브명' 형태이므로, 해당 형태로 반복되는 중복 판매 내역은 하나로 취급
